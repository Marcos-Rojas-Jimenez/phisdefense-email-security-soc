# Privacidad y sanitización

## Objetivo

Este repositorio está preparado para portfolio profesional, por lo que no contiene información sensible del servidor original ni datos privados innecesarios.

El objetivo es demostrar metodología, arquitectura, evidencias técnicas y resultados sin exponer secretos, credenciales o información sensible.

## Información eliminada o anonimizada

Para la versión pública del proyecto se anonimizaron o eliminaron:

- direcciones IP;
- rutas internas del servidor;
- sesiones;
- correos externos;
- posibles datos personales;
- configuraciones sensibles;
- claves privadas;
- credenciales;
- ficheros `.env`;
- tokens;
- backups completos del servidor.

## Dataset público

El archivo:

```text
data-samples/bateria_pruebas_soc_sample.csv
```

contiene una versión sanitizada del dataset SOC original.

Este dataset conserva:

- identificadores de prueba;
- tipo de evento;
- clasificación;
- resultado esperado;
- ubicación final;
- observaciones resumidas sanitizadas.

## Evidencias públicas

Las evidencias incluidas en este repositorio son extractos anonimizados.

Las evidencias completas originales se conservan fuera del repositorio público.

## Motivo

El objetivo del portfolio es mostrar capacidades técnicas en:

- Blue Team;
- SOC;
- seguridad de correo;
- análisis de logs;
- automatización;
- documentación de evidencias;
- desarrollo de dashboards.

No se busca exponer datos operativos sensibles del entorno original.

## Nota final

El servidor original utilizado durante el TFM puede desaparecer en el futuro.

Este repositorio mantiene una versión segura, documentada y sanitizada del proyecto.
