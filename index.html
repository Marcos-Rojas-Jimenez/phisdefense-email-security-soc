Que problema resuelve

En muchos entornos, la seguridad de correo se configura pero no se valida con evidencias. Este proyecto convierte controles como SPF, DKIM, DMARC y TLS en eventos medibles, documentados y visibles desde una perspectiva SOC.

Validacion defensiva

Se ejecutaron pruebas controladas para comprobar autenticacion de correo, rechazo de abuso SMTP, transporte seguro y proteccion ante suplantacion.

Evidencias reales

Las pruebas se apoyaron en logs, respuestas SMTP, consultas tecnicas, milter-reject, validaciones TLS y eventos detectados por Postfix/OpenDMARC.

Visualizacion SOC

Los eventos se transformaron en un dataset SOC para alimentar un dashboard en Python/Dash con metricas, categorias, tablas y evidencias.

Arquitectura del proyecto

El laboratorio conecta servicios reales de correo, logs del sistema, procesamiento automatizado y visualizacion ejecutiva.

Flujo de datos

  • Internet envia trafico SMTP al servidor.
  • Postfix, OpenDMARC, OpenDKIM y Dovecot generan eventos.
  • Syslog conserva evidencias tecnicas.
  • Scripts SOC procesan los eventos relevantes.
  • El CSV SOC alimenta el dashboard.
  • El dashboard muestra KPIs, tablas, categorias y evidencias.

Controles validados

  • SPF para autorizacion de origen.
  • DKIM para integridad y firma.
  • DMARC p=reject para rechazo de spoofing.
  • STARTTLS SMTP e IMAPS TLS para transporte seguro.
  • Rechazo de relay no autorizado.
  • Deteccion de auth_fail y recipient_unknown.
Nota: el servidor original puede no estar activo en el futuro. Este repositorio conserva una version documentada y sanitizada del proyecto para portfolio profesional.
